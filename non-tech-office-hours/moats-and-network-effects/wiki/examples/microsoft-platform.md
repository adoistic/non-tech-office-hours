# Microsoft — Platform Network Effects

**Concept:** [[platform-network-effects]]
**Source:** raw/nfx-network-effects-manual.md

## What they built

Microsoft built Windows as the dominant PC operating system from the 1980s onward, then layered Office as the dominant productivity suite. The Windows platform attracted third-party software developers who built applications only available on Windows; those applications attracted users; user density attracted more developers. Microsoft also gave Windows to universities for free, ensuring new graduates entered the workforce standardized on it.

## The mechanism

- Developers built Windows-only applications because Windows had the most users; users bought Windows PCs because they had the most applications — a reinforcing two-sided loop that grew as each side expanded.
- Positive direct same-side network effects: Windows users could share files with colleagues without conversion friction, making organizational standardization on Windows self-sustaining.
- Enterprise sales force locked in corporate contracts that made Windows the organizational default, adding institutional switching costs on top of the network effect.

## Why the analog often fails

Founders citing Microsoft as a platform comparable almost always underweight how long and expensive the platform build took. Microsoft cultivated the developer ecosystem over a decade with dedicated tools, SDKs, conferences, and university partnerships before the network effect was self-sustaining. A startup that builds an API and calls it a "platform" has not built a platform network effect; it has built an integration layer. The critical test is whether third-party developers build products exclusively for or primarily on the founder's platform. If developers also publish on Android, iOS, or a competitor simultaneously at no cost, the platform network effect is weak or absent.
