# Network Economies

**Category:** Helmer Power
**Source:** raw/yc-helmer-7-powers-transcript.md

## Definition

A network economy exists when the value of a product increases as more users or participants join. As Diana Hu summarizes from the book: "the value of the product increases as more users or customers get and use the product, and everyone derives more value as an effect of more people using it." Classic examples include Facebook (more friends = more value per user) and the Visa network (more merchants = more value per cardholder). Helmer's "network economies" is the same phenomenon NFX calls "network effects" — see [[network-effects]] for the 16-type taxonomy.

## Mechanism

- Each new node (user, merchant, data point) adds value to every existing node, creating a compounding advantage that scales non-linearly.
- In the AI era, the dominant expression is data: more usage generates more data, which improves the model, which improves the product, which attracts more users.
- Evals compound this loop — more real-world usage yields higher-quality feedback signals, which tighten the feedback loop faster than any competitor starting from zero.

## When it applies

- The product's value is demonstrably higher for existing users when new users join (not just higher revenue for the company).
- Data generated by usage can be fed back into model improvement in a closed loop.
- The network has a natural aggregation point: a platform, a marketplace, or a shared model that all users contribute to and all users benefit from.

## Kill-test

Name one specific user. How measurably better did their experience get the week your network added its last thousand users? If you cannot name the mechanism, you have a growth story, not a network economy.

## AI-native erosion check

Network economies are strengthened in the AI era, not eroded. Data network effects are the flagship AI-native moat: Cursor feeds every keystroke into model training; ChatGPT's conversation history informs successive training runs; enterprise AI agents accumulate proprietary workflow data that improves evals over time. The risk is fake claims — LLM personalization that improves per-user experience without improving value for other users mimics the shape without the substance.
