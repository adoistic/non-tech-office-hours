# Network Effects

**Category:** Umbrella — NFX taxonomy
**Source:** raw/nfx-network-effects-manual.md

## Definition

A network effect exists when each additional user makes the product more valuable for every existing user. More users → more value per user. This is a structural property of the business, not a growth strategy. Helmer's "network economies" names the same phenomenon — see [[network-economies]].

## Mechanism

- Each new node adds value to existing nodes, so the network compounds: value scales faster than headcount.
- The compounding creates a self-reinforcing loop — more value attracts more users, which adds more value.
- Distinguish from viral growth: virality drives acquisition speed; network effects drive retention and value. A product can be viral without any network effect, and vice versa. NFX is explicit: conflating the two is the most common founder mistake.

## When it applies

- Each new user measurably increases value for existing users — not just "more revenue for the company."
- The value increase is felt by users directly (not just inferred by the founder).
- Removing users from the network would degrade the product, not just shrink the market.

## Kill-test

Name three users currently on your network. How much more valuable did the product become for user A the day users B and C joined? If you cannot name the users and the specific mechanism, you do not have a network effect.

## AI-native erosion check

Most "network effect" claims in 2026 are fake. LLM-personalized feeds, recommendation systems, and "the more you use it the better it gets" loops mimic the surface shape of a network effect without the underlying mechanism — each new user does not increase value for existing users. The bar for a real network effect has risen, even though genuine ones (especially data network effects) are if anything strengthened in the AI era. The agent's job is to apply the kill-test rigorously and refuse the fake claims.
